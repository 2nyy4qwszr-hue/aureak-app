### [WARNING] — `unconfirmed` absent du DB enum attendance_status

**Fichier** : `page.tsx:320` et `page.tsx:836`
**Description** : Le code filtre `advancedStatuses = ['late', 'trial', 'injured', 'unconfirmed']`. Or la migration `00002_create_enums.sql` définit `attendance_status` avec seulement `present | absent | injured | late | trial` (5 valeurs). Le type TS `AttendanceStatus` dans `@aureak/types/enums.ts` ajoute `'unconfirmed'` qui n'existe pas en DB. Désynchronisation DB↔TS.
**Impact** : `'unconfirmed'` ne peut jamais arriver depuis la DB → filtre inoffensif mais le type TS est incorrect. Toute tentative d'upsert avec `status: 'unconfirmed'` échouerait silencieusement côté Supabase (constraint violation).
**Correction suggérée** : Soit ajouter `'unconfirmed'` au DB enum via migration, soit le retirer du type TS. À traiter dans une story dédiée (migration requise).
