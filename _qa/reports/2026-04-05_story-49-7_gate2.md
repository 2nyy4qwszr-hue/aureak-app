### Edge Cases Métier

**Saison inactive (aucune saison `is_current = true`)** :
La CTE `effective_season` retourne 0 lignes. Le JOIN filtre tout. La vue retourne 0 lignes.
`.maybeSingle()` retourne `data: null`. `getChildCurrentClubFromHistory` retourne `null`.
`autoClub` reste `null`. Affiche "Non renseigné pour la saison courante". ✅ SAFE

**Plusieurs saisons `is_current = true` (anomalie)** :
`DISTINCT ON (tenant_id) … ORDER BY tenant_id, start_date DESC` sélectionne la plus récente.
Comportement déterministe. ✅ SAFE

**childId invalide ou non existant** :
Vue retourne 0 lignes → `null` → affiche "Non renseigné". ✅ SAFE

**Joueur sans historique pour la saison courante** :
JOIN ne matche pas → vue retourne 0 lignes pour ce child_id → `null` → "Non renseigné". ✅ SAFE

**Erreur réseau / vue SQL non existante** :
`catch (err)` → console.error (dev) → `finally { setAutoClubDone(true) }` → `autoClub` reste
`null` → "Non renseigné pour la saison courante" affiché. Pas de crash de la page. ✅ SAFE

**Navigation rapide entre fiches joueurs** :
`useEffect(() => { ... }, [childId])` se re-déclenche à chaque changement de `childId`.
`setAutoClubDone(false)` en début d'effect masque la section pendant le nouveau chargement. ✅
Risque minimal de setState sur composant démonté (fonctions async sans AbortController), mais
l'impact est nul en pratique puisque la route est dynamique et le composant reste monté.

**`clubNom` : cast unsafe ?** :
`child_directory_history.club_nom` est TEXT NOT NULL (migration 00046 ligne 15). La vue ne
peut retourner null pour ce champ. Cast `as string` sûr. ✅

### [WARNING] W4 — Absence de guard `isMounted` pour setState async
`page.tsx:1486-1497` : l'IIFE async setAutoClub/setAutoClubDone peut s'exécuter après
démontage du composant si l'utilisateur navigue très rapidement. React 18 absorbe ce warning
silencieusement mais c'est une imperfection connue.
```typescript
// Pattern recommandé (non implémenté)
let active = true
;(async () => {
  try { const r = await getChildCurrentClubFromHistory(childId); if (active) setAutoClub(r) }
  catch { ... } finally { if (active) setAutoClubDone(true) }
})()
return () => { active = false }
```
Non bloquant pour cette story — pattern pré-existant dans le fichier pour d'autres useEffects.

### [WARNING] W5 — `saison` non affiché dans l'UI
La réponse contient `autoClub.saison` (ex: '2025-2026') mais ce champ n'est pas affiché.
L'admin ne peut pas vérifier visuellement que le club correspond bien à la saison courante sans
cliquer sur le lien. Suggestion : ajouter `({autoClub.saison})` après le nom de club.
Non bloquant — la story ne l'exige pas explicitement.
