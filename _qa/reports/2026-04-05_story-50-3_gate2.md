# QA Gate 2 — Story 50.3 : Dashboard Prochaine Séance Countdown Tile
Date: 2026-04-05

## Edge Cases & Bug Hunter

### Edge Case 1 — Aucune séance dans les 24h (AC2)
- `listNextSessionForDashboard()` retourne `{ data: null, error: null }` → `session=null`
- `CountdownTile` affiche "Aucune séance dans les 24h" + "Tout est calme ✓" ✅
- useEffect countdown ne démarre pas (guard `if (!session) return`) ✅

### Edge Case 2 — Séance dans le passé / countdown = 0 (AC4)
- `Math.max(0, ...)` empêche les valeurs négatives ✅
- `isOngoing = session !== null && secondsLeft === 0` → affiche "🟢 En cours" ✅
- `setInterval` continue à tourner (secondsLeft reste 0) — comportement correct ✅

### Edge Case 3 — session.location undefined (AC3)
- `location?: string` dans `UpcomingSessionRow` — rendu conditionnel `{session.location && ...}` ✅

### Edge Case 4 — Démontage avant réponse API
- `setLoadingUpcoming(false)` dans finally, pas de setState après unmount risk minime (fetch simple) — acceptable ✅

### Edge Case 5 — Changement de session.scheduledAt
- Dépendance useEffect : `[session?.scheduledAt]` → timer recalculé si la séance change ✅

### Edge Case 6 — groupName absent
- `(session['groups'] as { name?: string } | null)?.name ?? '—'` — fallback '—' ✅
