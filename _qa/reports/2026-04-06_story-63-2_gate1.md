## Issues Détectées

### [BLOCKER] — `mapStage` dans `academyStatus.ts` : fallback de statut invalide `'draft'`

**Fichier** : `aureak/packages/api-client/src/academy/academyStatus.ts:57`
**Description** : Le fallback `?? 'draft'` pour le champ `status` produisait une valeur qui n'est pas dans le type `StageStatus = 'planifié' | 'en_cours' | 'terminé' | 'annulé'`. La valeur `'draft'` n'existe pas dans cet enum TS ni en DB (la colonne a `DEFAULT 'planifié'`). Ce cast `as StageStatus` masque l'incohérence à la compilation mais produit un runtime value invalide si jamais la colonne est absente.
**Impact** : Crash silencieux ou comportement inattendu dans `STATUS_COLORS[event.status]` (undefined → `undefined + '20'` = `'undefined20'` pour la couleur) si ce mapStage est utilisé dans un chemin affichant le statut.
**Correction appliquée** :

```typescript
// avant
status : ((r.status as string | null) ?? 'draft') as import('@aureak/types').StageStatus,

// après
status : ((r.status as string | null) ?? 'planifié') as import('@aureak/types').StageStatus,
```

**Statut** : ✅ Corrigé automatiquement par l'agent QA.

### [WARNING] — `StatusBadge` : styles inline entièrement hardcodés hors tokens

**Fichier** : `aureak/apps/web/app/(admin)/evenements/page.tsx:58-60`
**Description** : Le composant `StatusBadge` utilise des valeurs numériques hardcodées en style inline : `paddingHorizontal: 9`, `paddingVertical: 2`, `borderRadius: 20`, `fontSize: 10`, `fontWeight: '700'`, et le suffixe de couleur `'20'` (hex 32 ≈ 12.5% opacité, différent du pattern `'1f'` utilisé partout ailleurs).
**Impact** : Incohérence visuelle potentielle, dette de maintenance.
**Correction suggérée** :
```tsx
// Extraire vers StyleSheet et utiliser space.* + radius.*
<View style={[s.statusBadge, { backgroundColor: color + '1f', borderColor: color }]}>
  <Text style={[s.statusBadgeText, { color }]}>{status.toUpperCase()}</Text>
</View>
// Dans StyleSheet :
statusBadge    : { borderWidth: 1, borderRadius: 20, paddingHorizontal: space.sm, paddingVertical: 2 },
statusBadgeText: { fontWeight: '700', fontSize: 10 },
```

**Fichier** : `aureak/apps/web/app/(admin)/evenements/page.tsx` — lignes 75, 95, 102-110, 317, 372, 379-381, 390-394, 407, 416, 420-422, 430, 459, 464, 469, 477
**Description** : Nombreux `borderRadius`, `fontSize`, `paddingHorizontal`, `paddingVertical`, `gap`, `height`, `width` hardcodés en valeurs numériques (7, 8, 9, 10, 11, 12, 14, 20, etc.). Le codebase utilise `space.*` et `radius.*` — ces valeurs devraient passer par les tokens.
**Impact** : Dette de style — cohérence design rompue si les tokens sont mis à jour.
**Note** : L'agent de code review interne (Dev Agent Record dans la story) a corrigé les `rgba()` hardcodés → `color + '1f'`, mais n'a pas traité les valeurs numériques de layout.

**Fichier** : `aureak/packages/api-client/src/academy/academyStatus.ts:217`

**Correction suggérée** : Renommer en `listAcademyStages()` ou la supprimer si elle peut être remplacée par la version `admin/stages.ts`.

### [WARNING] — AC6 partiellement implémenté : stub sans message explicite post-sélection

**Fichier** : `aureak/apps/web/app/(admin)/evenements/page.tsx:219-226`
**Description** : AC6 demande d'afficher "Création bientôt disponible" **après** la sélection du type non-stage dans le modal. **Correction suggérée** : Ajouter un second step dans le modal (ou un toast) qui affiche le message "bientôt disponible" directement après la sélection dans la modale, avant de rediriger.
