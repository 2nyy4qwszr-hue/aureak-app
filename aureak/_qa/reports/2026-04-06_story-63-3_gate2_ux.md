## Findings

### [WARNING] W-UX-1 — `DevSectionCard` recréé localement au lieu d'utiliser `@aureak/ui`

**Fichier** : `developpement/page.tsx` lignes 40-52

**Observation** : Le composant `DevSectionCard` (icône + titre + description + CTA "Voir →") est une `Card` avec contenu navigable. Le `@aureak/ui` dispose d'un composant `Card` avec variants `light/dark/gold`. L'implémentation en `Pressable` local est fonctionnellement correcte mais contourne le composant partagé.

**Nuance** : Le composant UI `Card` de `@aureak/ui` n'est pas connu pour avoir une prop `onPress` ou un variant navigable. Cette réimplémentation locale peut être justifiée si `@aureak/ui/Card` ne supporte pas l'interaction `Pressable`. Non bloquant pour une story de pages statiques.

**Sévérité** : WARNING (non bloquant — composant local justifiable pour un pattern de navigation card)

---

### [WARNING] W-UX-2 — fontSize hardcodés (non via `typography.*` tokens)

**Fichiers** : tous les 4 fichiers `.tsx`

**Valeurs trouvées** :

| Fichier:style | Valeur | Token typography disponible |
|---------------|--------|----------------------------|
| `page.tsx` title | `fontSize: 28` | `typography.h1.size = 28` |
| `page.tsx` sub | `fontSize: 15` | `typography.body.size = 15` |
| `page.tsx` cardTitle | `fontSize: 18` | `typography.h3.size = 18` |
| `page.tsx` cardDesc | `fontSize: 13` | `typography.bodySm.size = 13` |
| `page.tsx` cardLink | `fontSize: 13` | `typography.bodySm.size = 13` |
| `page.tsx` cardIcon | `fontSize: 36` | `typography.display.size = 36` |
| `prospection/page.tsx` title | `fontSize: 24` | Non couvert (entre h1=28 et h2=22) |
| `prospection/page.tsx` kpiValue | `fontSize: 32` | `typography.stat.size = 24` (divergence intentionnelle) |

**Observation** : Le projet utilise `StyleSheet.create` avec React Native Web, où le pattern `typography.h1.size` n'est pas directement injectié dans `StyleSheet`. L'usage de valeurs numériques directes conformes aux tokens (même valeurs) est un pattern acceptable dans ce contexte. Les valeurs sont toutes cohérentes avec les tokens définis.

**Sévérité** : WARNING (non bloquant — valeurs conformes aux tokens même si non importées via `typography.*`)
