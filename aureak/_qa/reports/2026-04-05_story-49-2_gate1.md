### BLOCKER-2 — `handleOpenThemePicker` sans try/catch ni état de chargement
**Fichier** : `page.tsx` lignes 190–197 (avant correction)
**Règle violée** : try/finally obligatoire sur tout state setter de chargement ; erreur silencieuse si `listMethodologyThemes` échoue
**Problème** :
- `listMethodologyThemes()` appelée sans `try/catch` → erreur réseau non gérée
- Aucun indicateur de chargement pendant le fetch des thèmes disponibles
- En cas d'erreur, `setShowThemePicker(true)` aurait pu s'exécuter avec `availableThemes` vide

**Correction appliquée** : ✅
```typescript
const handleOpenThemePicker = async () => {
  setThemeAddError(null)
  if (availableThemes.length === 0) {
    setAddingTheme(true)
    try {
      const all = await listMethodologyThemes({ activeOnly: true })
      setAvailableThemes(all)
    } catch (err) {
      if (process.env.NODE_ENV !== 'production') console.error('[SessionDetail] listMethodologyThemes error:', err)
      setThemeAddError('Impossible de charger la liste des thèmes.')
      return
    } finally {
      setAddingTheme(false)
    }
  }
  setShowThemePicker(true)
}
```
