## 2. Findings

### [MEDIUM — CORRIGÉ] M1 — Couleurs rgba hardcodées dans les bannières (3 fichiers)

**Fichiers** :
- `developpement/prospection/page.tsx:63,65`
- `developpement/marketing/page.tsx:63,65`
- `developpement/partenariats/page.tsx:63,65`

**Problème** : Les styles de `banner` utilisaient `'rgba(193,172,92,0.10)'` (backgroundColor) et `'rgba(193,172,92,0.25)'` (borderColor) en dur. Le `borderColor` correspond exactement à `colors.border.gold` (token existant). Le `backgroundColor` à 0.10 n'avait pas de token dédié.

**Règle violée** : CLAUDE.md — "Styles UNIQUEMENT via `@aureak/theme` tokens — jamais de couleurs/espacements hardcodés".

**Correction appliquée** :
1. Ajout du token `colors.border.goldBg = 'rgba(193,172,92,0.10)'` dans `packages/theme/src/tokens.ts`
2. Remplacement dans les 3 fichiers :
   - `backgroundColor: 'rgba(193,172,92,0.10)'` → `colors.border.goldBg`
   - `borderColor: 'rgba(193,172,92,0.25)'` → `colors.border.gold`

- **KPI value "—"** : Valeur em-dash (`—`) correctement alignée center avec `fontWeight: '900'` — lecture immédiate que la donnée est vide, pas zéro.
- **Absence de `accessibilityLabel`** sur les `Pressable` du hub — observation mineure, non bloquant pour un stub.
